## 1. プロジェクト概要

街を舞台にしたランニング × 動物保護ゲーム。プレイヤー(子ども)とのアイデア出しから生まれた個人開発プロジェクト。

**コアコンセプト**
- 街を自動走行するキャラクターをタップでジャンプ操作
- 道に現れる動物(犬・猫)は「タップして助ける」か「ジャンプしてよける」かを選べる二択ギミック
- 助けた動物は「どうぶつの国」に集まり、なでる/ごはん/おうちの色を変える、という簡易シム要素で愛着を持って遊べる

**対象ユーザー**: 子ども〜ファミリー層。操作はワンタップ中心、文章はやさしい日本語(漢字少なめ)。

## 3. 技術スタック

- 単一HTMLファイル(`index.html`)に HTML / CSS / JavaScript(Vanilla JS, ES5相当の書き方)を内包
- 外部ライブラリ・外部CDN依存なし
- 効果音・BGMは **Web Audio API のoscillatorで全て合成**(音声ファイル一切なし → 著作権リスクゼロ、軽量、即ロード)
- フォントはシステムフォント依存(`Hiragino Maru Gothic ProN` 等)、Webフォント読み込みなし

このスタックを選んだ理由: 開発者はCLI・ローカル環境を使わず、Claudeとのチャット上でのvibe codingのみで開発する運用のため、依存を極力減らし「1ファイルで完結」する構成にしている。

## 4. ゲームデザイン詳細

### 4.1 画面構成
4画面をJSで切り替え(`showScreen()`)、CSSクラス`.active`の付け外しで表示制御。

1. **スタート画面** (`#screen-start`) — タイトル、あそぶボタン、どうぶつの国ボタン
2. **ゲーム画面** (`#screen-game`) — メインプレイ。HUD(ハート/コイン/動物数/音量ボタン)+ ゲームエリア
3. **ゲームオーバー画面** (`#screen-gameover`) — 結果表示、リトライ/どうぶつの国への導線
4. **どうぶつの国画面** (`#screen-kingdom`) — 保護した動物の一覧・育成

### 4.2 コアメカニクス

- **自動スクロール+タップジャンプ**: プレイヤーは画面左寄り固定位置(`left:60px`)。アイテムは右から左へ流れてくる。画面タップで`jump()`発火、CSSアニメーション(`jumpArc`, 0.55秒)で上下動。
- **当たり判定ゾーン方式**: プレイヤー位置を中心に`ZONE_LEFT`〜`ZONE_RIGHT`の判定帯を定義。物理エンジンは使わず、アイテムのx座標がこの帯を通過するタイミングで判定する軽量実装。
- **動物アイテムの3パターン**(`updateItems()`内で分岐):
  1. ゾーン内でタップ → `rescueAnimal()` で即座に保護、どうぶつの国へ追加
  2. ゾーン内でジャンプ中(`isJumping===true`) → `status='safe'` としてノーダメージで通過
  3. 何もせずゾーンを通過 → `registerHit()` でハート−1、0.9秒の無敵時間発生
- **コイン/スター**: ゾーン内に入った時点で自動加算(ジャンプ・タップ不要)。`item.x`がプレイヤー判定帯に入るだけで回収される。
- **難易度上昇**: `state.speed = min(3.0 + score*0.025, 5.5)` でスコアに応じてスクロール速度が緩やかに上昇(上限あり)。
- **ライフ**: 3ハート制。0になったら`gameOver()`。

### 4.3 どうぶつの国

- 保護した動物はカード形式で一覧表示(2列グリッド)
- 各カードに3アクション: 🖐️なでる / 🍖ごはん(どちらもhappiness+1、上限5) / 🎨おうちをかざる(背景色を6色からローテーション)
- happinessは❤️/🤍の5段階表示
- 現状、保存機能なし(リロードでリセット。詳細は「6. 既知の課題」参照)

## 5. キャラクターデザインについて(経緯と判断)

**変更前**: プレイヤーキャラを絵文字`🧒`一文字で表現していた(プロトタイプ初期版)。

**指摘・判断**: 「自分で作ったキャラクター」というゲームの核となるアイデアに対して、汎用絵文字1文字は表現として弱く、静止画のままで動きの説得力も無かった。これを受けて、CSSのdivレイヤーを組み合わせた自作キャラクターに置き換え済み(現行版)。

**現行の実装方法**:
- `#player`内に`.p-hair` `.p-head` `.p-body` `.p-arm-l/r` `.p-leg-l/r`の7パーツをdivで構成
- 走行中(`.running`クラス付与時)は脚・腕を左右逆位相でCSS keyframesアニメーション(`legSwingL/R`, `armSwingL/R`)させてランニングモーションを表現
- ジャンプ中は脚・腕を「タック」した静止ポーズに切り替え、躍動感を演出
- 被弾時は`brightness`+`drop-shadow`によるフラッシュ演出

**今後の拡張候補(未実装)**: 髪色・服の色・小物(帽子等)を選べるカスタマイズ機能を足すと、「自分だけのキャラクター」という当初コンセプトをより強く実現できる。実装難度は低〜中(CSS変数を動的に書き換える方式で対応可能)。

## 6. オーディオシステム

すべて **Web Audio API のoscillator合成音**。音声ファイルは一切使用していない(著作権・容量の問題を回避するため)。

| 種類 | 関数 | 内容 |
|---|---|---|
| ジャンプ | `seJump()` | square波、320→640Hzの上昇スイープ |
| コイン取得 | `seCoin()` | square波の2音階チャイム |
| 動物救助 | `seRescue()` | triangle波の3音上昇アルペジオ |
| 被弾 | `seHit()` | sawtooth波、220→90Hzの下降音 |
| ゲームオーバー | `seGameOver()` | triangle波の4音下降メロディ |
| ボタンタップ | `seTap()` | sine波の短いクリック音 |
| BGM | `playBgmStep()`他 | 8音の明るいループをsetTimeoutで自走再生(`BGM_NOTES`配列を編集すれば曲調変更可) |

**実装上の注意点**:
- ブラウザの自動再生制限により、`AudioContext`は初回のユーザー操作(タップ)まで起動しない。`document.body`への初回click一回限りのリスナーで`ensureAudio()` + BGM開始をしている。
- 全画面の右上に🔊/🔇トグルボタン(`.sound-btn`)を配置。`state.musicOn`フラグで一括制御。
- BGMは画面遷移を跨いで鳴り続ける仕様(ゲームオーバーで止めない)。ミュート操作でのみ停止。

## 7. 既知の課題 / TODO

- **永続化なし**: ハイスコア・どうぶつの国の状態はリロードで消える。今後Supabase等のバックエンドと繋げば永続化可能(他プロジェクトのSupabase基盤を流用できる可能性あり)。
- **キャラクターカスタマイズ未実装**(5章参照)
- **効果音/BGMの音量バランス**: 実機(特にiPhone Safari)での聞こえ方は要確認・微調整余地あり
- **当たり判定のタイミング調整**: `ZONE_LEFT/RIGHT`や`speed`の値は仮決め。実プレイでの体感を見て調整が必要
- **アクセシビリティ**: 音が出せない環境への配慮(視覚エフェクトのみで完結はしているので致命的ではない)
